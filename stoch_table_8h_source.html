20  /*! \class stochTable
21  * Stochastic table stores stochastic values in a vector. Data structure is
22  * implemented to reduce the memory that would be needed if we allocated a 2D
23  * table with all the transitions. This lookup table is 1D and only stores
24  * those values which we add. Each the value are stores
25  * (Current state, Previous State, Score as P(X)). In addition, the indices
26  * of each state and position are recorded so that they can be referenced
27  * quickly. So the memory size necessary to store the values for a sparsely
28  * connected model are greatly reduce. However, the time to recreate the table
29  * is increased because it doesn't allocate the memory up front, but rather
30  * pushes values as they are recorded.
31  */
