38 /*!
39  @mainpage StochHMM: A Flexible C++ Hidden Markov Model Library and Application
40 
41  StochHMM is a free, open source C++ Library and application that implements HMM
42  from simple text files. It implements traditional HMM algorithms in addition it
43  providing additional flexibility. The additional flexibility is achieved by
44  allowing researchers to integrate additional data sources and application into
45  the HMM framework.
46 
47  For documentation on model syntax and designing a model, see Github wiki.
48 
49  ###http://www.github.com/KorfLab/StochHMM/wiki
50 
51  ##Integrating Data
52  Here are a few of the ways that StochHMM allows the users to integrate additional
53  data sources:
54  1. Multiple Emission States
55  2. Weighting or Explicitly Defining State paths on a sequence
56  3. Linking States Emissions/Transitions to external user-defined functions
57 
58 
59  ##Multiple Emission States
60 
61  StochHMM allows the user to provide multiple sequences. These sequences are then
62  handled by the emissions. These sequences can be REAL numbers or discrete characters/words.
63  StochHMM allows each state to have many emissions (Discrete or Continuous). Discrete emissions
64  can be independent of each other or joint distributions. The continuous emissions
65  can be considered in multiple ways. 1) They can be considered as raw probabilities which
66  will be integrated without transformation. 2) They can be considered as values to be plugged
67  into a Univariate Probability Distribution Function or Multivariate PDF (In the case of multiple
68  REAL sequences.
69 
70  Each states emissions are user-defined, so one state may have emissions from two
71  different sequences, while another may only have a single emission from a single sequence.
72 
73  ##Weighting or Explicitly Defining State paths to follow on a sequence.
74 
75  Often, we have some prior knowledge about the sequence. If this is the case,
76  we may want to integrate that into the model, without redesigning or retraining the model (a timely endeavor).
77  StochHMM allows the user to explicitly define a State path (By name of state, or category of state).
78  In addition, StochHMM also allows the user to weight a states path (By name of State or category of state defined by user)
79  This allows the user to restrict the predicted path or weight their prior knowledge.
80 
81 
82  ##Linking States Emissions or Transitions to external user-defined functions
83 
84  When that transition/emission is evaluated the function is called and can provide an emission. While
85  this may provide one way of addressing a weakness of HMMs, which is that they do not handle long
86  range dependencies. We see it rather as a way to link together existing utilities or functions that
87  provide additional information to the decoding algorithms. In this way, we can link divergent
88  datasets or functions within the HMM trellis in order to arrive at a better prediction.
89

91  ##Brief list of features implemented in StochHMM:
92  - General settings within Hidden Markov Models
93  1. User-defined HMM model via simple human readable text file
94  2. User-defined Alphabet
95  3. User-defined Ambiguous Characters
96  - States
97  1. Emissions
98  - Multiple emission states (Discrete / Continuous)
99  - Independent (Single or Multiple Discrete)
100  - Joint Distribution (Multiple Discrete)
101  - Univariate PDF (Single Sequence - Continuous)
102  - Multivariate PDF (Multiple Sequence - Continuous)
103  - Linkable to user-defined function
104  2. Transitions
105  - Standard Transitions
106  - Lexical Transitions (Single or multiple emission)
107  - Explicit Duration Transitions
108  - Linkable to user-defined functions
109  - Decoding
110  1. Traditional Decoding Algorithms
111  - Forward/Backward/Posterior
112  - Viterbi
113  - N-best Viterbi
114  2. Stochastic Sampling Decoding Algorithms
115  - Stochastic Forward
116  - Stochastic Viterbi
117  - Stochastic Posterior
118  - Decoding Traceback Path output formats
119  - State Path Index
120  - State Path Label
121  - GFF
122  - Hit Table (Stochastic Algorithms)
